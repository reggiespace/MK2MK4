# Guided Creation Wizard + Algorithm-Aware Reels Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn `/ideate` into a binding, sequential wizard (Account → Style → Template → Topic → AI-write → AI-render → Review), and make reels algorithm-aware (structured script templates, burned-in captions, loop points), plus a soft archive for abandoned pieces and a recent-publications memory that keeps AI-suggested topics from repeating.

**Architecture:** Prisma schema grows a `ReelTemplate` enum, `ContentPiece.reelTemplate`, `PieceStatus.archived`, and `Slide.caption`. The LLM layer (`apps/web/src/lib/llm/`) gets a static reel-template registry and takes format/template as hard constraints instead of suggestions. The pieces-creation API stops trusting the LLM's own format opinion. The Ideate UI becomes a 4-step wizard on one route. The worker (`apps/worker/app/renderer/reel.py`) burns per-slide captions via ffmpeg's `subtitles` filter and adds a loop-out crossfade on the Ken Burns path.

**Tech Stack:** Next.js 16 (apps/web), Prisma 7 + Postgres, Zod, Vitest; Python FastAPI worker (apps/worker), Pillow, ffmpeg, pytest.

## Global Constraints

- Format enum: `single | carousel | reel | story` (Prisma `Format`, `apps/web/prisma/schema.prisma:89-94`) — unchanged.
- Image templates: `classic | editorial_bold | bold_highlight | minimal_card | photo_overlay` (Prisma `Template`) — unchanged, still used for single/carousel/story.
- Reel templates are a **new, separate** enum/field — never merged into `Template`.
- All Prisma migrations: run from `apps/web/` via `npx prisma migrate dev --name <name>`.
- Web tests: `npm test` (vitest) from `apps/web/`. Worker tests: `pytest` from `apps/worker/` (activate `.venv` first if not already on PATH: `source apps/worker/.venv/bin/activate`).
- Never let AI-generated imagery contain rendered text — already enforced in `apps/web/src/lib/imagery/prompt.ts` (`GUARDRAIL_SUFFIX`) and `apps/worker/app/renderer/imagery.py` (`NO_TEXT_SUFFIX`). Do not touch these guardrails.
- Commit after every task's tests pass — small, frequent commits, never batch multiple tasks into one commit.

${pillars}`;

- Frame everything as general education, not medical advice or a guarantee.
- Avoid language patterns common in weight-loss ad copy (dramatic claims, before/after phrasing, guarantees).`;

  brand: BrandContext,
  opts: {
    count: number;
    pillarName?: string;
    brief?: string;
    format?: string;
    recentPublications?: { title: string; angle: string }[];

${formatBlock}
${recentBlock}
Each idea: a punchy title, a one-sentence angle, a recommended format (single | carousel | reel | story), and the pillar it belongs to.
Reels suit step-by-step or "why this happens" explainers; carousels suit multi-point education; single images suit one strong stat or reframe; stories suit quick, timely reminders or a 1-3 frame tip.

  };

export function draftPrompt(
  brand: BrandContext,
  opts: {
    title: string;
    angle: string;
    format: string;
    reelTemplate?: ReelTemplateDef;
    story?: import("./types").StoryBrief;
  },
): { system: string; user: string } {
  const storyBlock = opts.story
    ? `\nStory to tell: ${opts.story.story}\nKey message: ${opts.story.keyMessage}\nBeats: ${opts.story.beats.join(" → ")}\nCTA intent: ${opts.story.ctaIntent}\n`
    : "";

  const reelStructureBlock =
    opts.format === "reel" && opts.reelTemplate
      ? `\nREEL SCRIPT STRUCTURE — "${opts.reelTemplate.label}" (follow exactly, ${opts.reelTemplate.beats.length} slides, in order):
${opts.reelTemplate.beats.map((b, i) => `${i + 1}. [${b.role}] ${b.instruction}`).join("\n")}

Loop: ${opts.reelTemplate.loopHint}
Each slide also needs a "caption" field: a short on-screen line (matching what the voiceover says for that beat) that will be burned into the video.
`
      : "";

  return {
    system: brandPreamble(brand),
    user: `Write a complete post for this idea:
Title: ${opts.title}
Angle: ${opts.angle}
Preferred format: ${opts.format}${storyBlock}${reelStructureBlock}

${HEALTH_CONTENT_GUARDRAILS}

Produce:
- caption: link-free caption (light tasteful emoji), value-first, no medical claims. End with a nudge, not a URL (EN "Try it free — links in the first comment 👇" / PT "É grátis pra testar — links no primeiro comentário 👇").
- firstComment: BOTH destinations + one post-specific question, exactly:
  "<EN: Everything's free to try 👇 / PT: Tudo grátis pra testar 👇>
  🌐 Web — gastric-iq.com
  📲 Android — play.google.com/store/apps/details?id=ca.reggiespace.gastric_iq
  <one short engagement question in the post's language>"
- hashtags: 4-8 relevant hashtags (no leading #).
- recommendedFormat: single | carousel | reel | story + formatRationale (one sentence) — informational only, the format above is already decided.
- slides: ordered slides.
  - "carousel": 1 cover (eyebrow + short headline), 2-4 body (eyebrow + headline + 1-2 sentence body), 1 cta (headline = call to action).
  - "single": exactly 1 cover (eyebrow + headline).
  - "reel": follow the REEL SCRIPT STRUCTURE above exactly — one slide per beat, each with a "caption" field, plus a "voiceover" field with the full narration covering every beat in order.
  - "story": 1-3 full-screen frames (eyebrow + short headline, optional 1-sentence body); last frame is a cta.

Respond as JSON matching:
{ "caption", "firstComment", "hashtags": [], "recommendedFormat", "formatRationale", "slides": [ { "role": "cover|body|cta", "eyebrow", "headline", "body", "caption", "imagePrompt": "short literal scene description for a background photo — food, ingredients, calm lifestyle objects; no people's bodies, no medical content, no text" } ], "voiceover" }`,
  };
}

// apps/web/src/lib/pipeline/recent-publications.ts
import "server-only";
import { prisma } from "@/lib/db";

/**
 * Bounded, cheap lookup of what a brand has recently scheduled/published, so
 * AI-suggested topics can avoid repeating the same angle. Not a growing
 * history table — always just the last `limit` pieces per brand.
 */
export async function getRecentPublications(
  brandId: string,
  limit = 20,
): Promise<{ title: string; angle: string }[]> {
  const pieces = await prisma.contentPiece.findMany({
    where: { brandId, status: { in: ["scheduled", "published"] } },
    orderBy: { createdAt: "desc" },
    take: limit,
    select: { idea: { select: { title: true, angle: true } } },
  });
  return pieces
    .filter((p): p is { idea: { title: string; angle: string } } => p.idea !== null)
    .map((p) => ({ title: p.idea.title, angle: p.idea.angle }));
}

"use client";

import { useState, useCallback } from "react";
import { useRouter } from "next/navigation";

type Pillar = { id: string; name: string; description: string };
type Brand = {
  id: string;
  name: string;
  locale: string;
  publisher: string;
  pillars: Pillar[];
  brandKit: { defaultSkin: string } | null;
};

type Step = "account" | "style" | "template" | "topic";

const STYLE_OPTIONS: { value: string; label: string; hint: string }[] = [
  { value: "carousel", label: "Carousel", hint: "Multi-slide, step-by-step education" },
  { value: "reel", label: "Reel", hint: "Short vertical video with voiceover" },
  { value: "single", label: "Single image", hint: "One strong stat or reframe" },
  { value: "story", label: "Story", hint: "Quick 1-3 frame reminder" },
];

const STEP_ORDER: Step[] = ["account", "style", "template", "topic"];

export function IdeateClient({
  brands,
  defaultBrandId,
  initialStep,
}: {
  brands: Brand[];
  defaultBrandId?: string;
  initialStep: Step;
}) {
  const router = useRouter();
  const [step, setStepState] = useState<Step>(initialStep);
  const [brandId, setBrandId] = useState(defaultBrandId ?? brands[0]?.id ?? "");

  return (
    <div className="page">
      <div className="page-header">
        <h1 className="page-title">Ideate</h1>
      </div>

        {STEP_ORDER.map((s, i) => (
          <span key={s} className={`wizard-step ${i === stepIndex ? "active" : i < stepIndex ? "done" : ""}`}>
            {i + 1}. {s[0].toUpperCase() + s.slice(1)}
          </span>
        ))}
      </div>

      {step === "account" && (
        <section className="section">
          <p className="eyebrow">Brand</p>
          <div className="brand-picker">
            {brands.map((b) => (
              <button
                key={b.id}
                type="button"
                className={`brand-card ${b.id === brandId ? "selected" : ""}`}
                onClick={() => {
                  setBrandId(b.id);
                  goToStep("style");

                <span className="muted">
                  {b.locale === "pt_BR" ? "pt-BR" : "EN"} · {b.publisher}
                </span>
              </button>
            ))}
          </div>
        </section>
      )}

      {step === "style" && selectedBrand && (
        <section className="section">
          <p className="eyebrow">Style</p>
          <div className="style-picker">
            {STYLE_OPTIONS.map((s) => (
              <button
                key={s.value}
                type="button"
                className={`style-option ${format === s.value ? "active" : ""}`}
                onClick={() => {
                  setFormat(s.value);
                  goToStep("template");
                }}
                title={s.hint}
              >
                <span className="style-option-label">{s.label}</span>
                <span className="muted sm">{s.hint}</span>
              </button>
            ))}
          </div>
          <button type="button" className="ghost sm" onClick={() => goToStep("account")}>
            ← Back
          </button>
        </section>
      )}

      {step === "template" && (
        <section className="section">
          <p className="eyebrow">Template</p>
          <p className="muted">Template step placeholder — implemented in Task 10.</p>
          <button type="button" className="ghost sm" onClick={() => goToStep("style")}>
            ← Back
          </button>
        </section>

      {step === "topic" && (

          <p className="eyebrow">Topic</p>
          <p className="muted">Topic step placeholder — implemented in Task 11.</p>

            ← Back
          </button>
        </section>

    </div>
  );
}

  const [pillarId, setPillarId] = useState<string | undefined>();
  const [brief, setBrief] = useState("");
  const [count, setCount] = useState(5);
  const [ideas, setIdeas] = useState<Idea[]>([]);
  const [loading, setLoading] = useState(false);
  const [drafting, setDrafting] = useState<string | null>(null);
  const [error, setError] = useState("");

  async function suggestIdeas() {
    setError("");
    setLoading(true);
    setIdeas([]);
    try {
      const res = await fetch("/api/ideas/suggest", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ brandId, pillarId, brief: brief || undefined, count, format }),
      });
      const data = await res.json();
      if (!res.ok) throw new Error(data.error ?? "Failed");
      setIdeas(data.ideas);
    } catch (e: unknown) {
      setError(e instanceof Error ? e.message : "Something went wrong");
    } finally {
      setLoading(false);
    }
  }

  async function draftFromIdea(ideaId: string) {
    setDrafting(ideaId);
    try {
      const res = await fetch("/api/pieces", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ ideaId, format, template, reelTemplate }),
      });
      const data = await res.json();
      if (!res.ok) throw new Error(data.error ?? "Draft failed");
      router.push(`/pieces/${data.piece.id}`);
    } catch (e: unknown) {
      setError(e instanceof Error ? e.message : "Draft failed");
      setDrafting(null);
    }
  }

Modify `_assemble` in `reel.py` to accept `slides` (needed for `_build_srt`) and `template`, write the SRT file, and append the `subtitles` filter to the final video stream. Replace the function signature and the tail of `_assemble` (lines 110-193):

```python
def _assemble(
    clips: list[Path],
    overlays: list[Path],
    audio_path: Path,
    motion: bool,
    durations: list[float],
    slides: list[dict[str, Any]],
    template: str,
) -> bytes:
    """Assemble slide clips/frames + audio + burned-in captions into MP4."""
    with tempfile.TemporaryDirectory() as tmp:
        tmp_path = Path(tmp)
        n = len(clips)

        if motion:
            filter_parts: list[str] = []
            inputs: list[str] = []
            for i, (clip_p, overlay_p) in enumerate(zip(clips, overlays)):
                inputs += ["-t", str(durations[i]), "-i", str(clip_p)]
                inputs += ["-i", str(overlay_p)]
                filter_parts.append(
                    f"[{i * 2}:v]scale=1080:1920:force_original_aspect_ratio=increase,"
                    f"crop=1080:1920,setpts=PTS-STARTPTS[bg{i}];"
                    f"[bg{i}][{i * 2 + 1}:v]overlay=0:0[v{i}]"
                )
        else:
            filter_parts = []
            inputs = []
            for i, clip_p in enumerate(clips):
                inputs += ["-loop", "1", "-t", str(durations[i] + FADE_DURATION_S), "-i", str(clip_p)]
                filter_parts.append(
                    f"[{i}:v]scale=1080:1920:force_original_aspect_ratio=increase,"
                    f"crop=1080:1920,"
                    f"zoompan=z='min(zoom+0.0015,1.05)':x='iw/2-(iw/zoom/2)':y='ih/2-(ih/zoom/2)'"
                    f":d={int(durations[i] * 25)}:fps=25:s=1080x1920[v{i}]"
                )

        if n > 1:
            xfade_chain = ""
            prev = "v0"
            cumulative = durations[0]
            for i in range(1, n):
                offset = cumulative - i * FADE_DURATION_S
                out_label = f"xf{i}" if i < n - 1 else "vraw"
                xfade_chain += (
                    f";[{prev}][v{i}]xfade=transition=fade:duration={FADE_DURATION_S}"
                    f":offset={offset}[{out_label}]"
                )
                prev = out_label
                cumulative += durations[i]
            filter_complex = ";".join(filter_parts) + xfade_chain
        else:
            filter_complex = filter_parts[0].replace("[v0]", "[vraw]")

        # Burn synced captions as a final filter stage on the merged stream.
        srt_text = _build_srt(slides, durations)
        video_map = "[vraw]"
        if srt_text.strip():
            srt_path = tmp_path / "captions.srt"
            srt_path.write_text(srt_text, encoding="utf-8")
            # ffmpeg filter args need colons/backslashes escaped in the file path.
            escaped_path = str(srt_path).replace("\\", "\\\\").replace(":", "\\:")
            style = _caption_style_for_template(template)
            filter_complex += f";[vraw]subtitles={escaped_path}:force_style='{style}'[vout]"
            video_map = "[vout]"
        else:
            filter_complex = filter_complex.replace("[vraw]", "[vout]")
            video_map = "[vout]"

        audio_idx = n * 2 if motion else n
        out_path = tmp_path / "reel.mp4"
        cmd = (
            ["ffmpeg", "-y"]
            + inputs
            + ["-i", str(audio_path)]
            + [
                "-filter_complex", filter_complex,
                "-map", video_map,
                "-map", f"{audio_idx}:a",
                "-c:v", "libx264", "-preset", "fast", "-crf", "23",
                "-c:a", "aac", "-b:a", "128k",
                "-shortest",
                "-pix_fmt", "yuv420p",
                str(out_path),
            ]
        )
        subprocess.run(cmd, check=True, capture_output=True)
        return out_path.read_bytes()
```
